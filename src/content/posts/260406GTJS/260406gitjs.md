## 🥑Git Clone加速：一行命令配置全局镜像

>除了网页端访问，开发者通过Git命令Clone仓库时，可配置全局镜像，无需每次手动修改链接，实现“ 一次配 置，永久加速” 。

1. 配置gitclone.com全局镜像

- 打开命令提示符（Windows：Win+R输入 cmd ；Mac/Linux：打开终端）；

-  输入以下命令并回车：

```cmd
git config --global url."https://gitclone.com".insteadOf https://
```

3. 配置完成后，直接用原GitHub链接Clone，Git会自动通过镜像站加速：
```cmd
git clone https://github.com/tensorflow/tensorflow.git
```

## 🍑使用DNS加速

>这个方法测试下来不太稳定，还是镜像站稳定点，当然如果前面的镜像站你都没法使用，可以尝试下这个方式

>[!warning]
此方法需具备一定操作能力，请勿轻易尝试！
### 查询GitHub有效IP地址
- 打开浏览器，访问站长工具DNS查询页面：http://tool.chinaz.com/dns/

### 修改系统Hosts文件


1. 打开电脑本地文件夹，路径为：`C:\Windows\System32\drivers\etc`

2. 找到名为`hosts`的文件（无后缀名），右键选择“ 用记事本打开” （如果提示权限不足，可先右键选择“ 属性” -“ 安全” -“ 编辑” ，赋予当前用户修改权限）

3. 滚动到hosts文件的最后一行，另起一行输入以下内容（IP地址和域名之间用至少一个空格隔开）：`20.205.243.166 github.com`

4. 输入完成后，按Ctrl+S保存文件，然后关闭记事本


### 刷新DNS解析缓存

1. 按下键盘上的<kbd>Win+R</kbd>组合键，打开运行窗口
2. 在输入框中输入“cmd”，按下回车键，打开命令提示符窗口
3. 在命令窗口中输入命令：ipconfig/flushdns
4. 按下回车键执行命令，当出现“Windows IP配置 已成功刷新DNS解析缓存”的提示时，说明操作成功
