## Umami博客数据统计系统
>Umami 是一个开源、注重隐私的网站分析工具，可以替代 Google Analytics更加方便快捷的帮助您分析博客的浏览数据，看看都有谁在偷偷关注你！。

**⚠️**本教程仅适用于<mark>Mizuki 8.2(1dcaa61)</mark>或更高版本，其他站点请移步官方文档。
### Umami的配置
1. 注册|登录[Umami](https://cloud.umami.is/)
2. 创建一个新的网站，获取网站ID
   - Add website-创建新的站点 
   - Name: yuqi- 网站名称
   - Domain：https://yuqi.ml/ - 网站域名
   - 点击Save- 创建站点
3. **必要信息获取**：
   - [❌]进入站点控制台即可看到网站ID:*Website Id*
   ```
   35XXXX97-d7f3-43Xb-a399-1caXXXe9fc2d
   注意以上仅做参考，以实际为准
   ```
   - [✔️]**跟踪脚本地址:*racking Script URL***
   ```
   <script defer src="https://cloud.umami.is/script.js" data-website-id="35xxxx97-d7f3-43xb-a399-1caxxxe9fc2d"></script>
   ```
   - [✔️]**umami统计分享链接:*shareURL***<br>
`Add`——`输入项目名称`——`Save`
   ```
   https://cloud.umami.is/share/t9TZAPPlPqj8q4PS
   可点击访问实验
   ```
### Mizuki配置
1. 配置`astro.config.mjs`
- 打开`astro.config.mjs`文件，找到`umami`集成配置项：
```
astro.config.mjs
export default defineConfig({
	site: siteConfig.siteURL,
	base: "/",
	trailingSlash: "always",

	output: "static",
   <!-- 大概位于40-43行 -->
	integrations: [
		umami({
			shareUrl:"https://cloud.umami.is/analytics/us/websites/27axxxx0-db09-4xx8-b3c7-8e6xxxx59x26", shareUrl: "YOUR_SHARE_URL", // Umami 分享链接（见下方说明）如设置为 false 则禁用组件的umami访问量信息显示,不影响umami统计
		}),

    // ... 其他集成
  ],
});
```
⭕Umami Cloud 用户注意

- 如果您使用的是 Umami Cloud（官方托管服务），您将得到：
```
https://cloud.umami.is/share/xxxxx
```
- 此时不能直接使用此链接！ 您需要：
点击该链接直至重新加载网页，复制地址栏中的链接，然后将其作为 shareUrl 的值。
```
https://cloud.umami.is/analytics/us/share/xxxxxxxxx
以实际链接为准
```
2. 配置`Layout.astro`
- 打开`src/layouts/Layout.astro`文件，在 \<head\> 标签内添加 Umami 跟踪脚本：
   - 对于官方托管服务的用户：
    ```
    <!-- 大约位于156-170行 -->
    <head>
    <!doctype html>
    <html
	 lang={siteLang}
	 class="bg-[var(--page-bg)] text-[14px] md:text-[16px]"
	 data-overlayscrollbars-initialize
    >
	 <head>
		<meta charset="UTF-8" />
		<!-- Umami -->
        <script defer src="https://cloud.umami.is/script.js" data-website-id="27a68670-db09-4178-b3c7-8e6fa5d59126"></script>
		
		<!-- 其他 -->
		
    ```
   - 对于自建Umami服务的用户：
   ```
   <script defer src="https://your-umami-instance.com/script.js" data-website-id="your-website-id"></script>
   请自行替换为实际服务地址
   ```
### 配置完成
1. 重新构建您的本地项目``pnpm build``
2. 部署您的网站到服务器上
3. 打开您的网站，查看 Umami 统计信息是否正常显示
### 出现问题
- 确认 shareUrl 配置正确（Umami Cloud 用户请确保使用重定向后的链接）
- 检查跟踪脚本地址是否正确
- 查看浏览器控制台是否有错误信息
- 确认 Umami 服务是否正常运行
